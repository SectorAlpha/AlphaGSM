AlphaGSM changelog
==================

Current release notes should be updated in every PR that changes user-facing behaviour,
server support, CI, packaging, or workflow defaults.

Historical detail lives in git history. Keep this file focused on release-facing notes and
high-level milestones.

2026-04-30
- CLI: add a top-level `self-update` command that checks for newer AlphaGSM releases, fast-forwards tracked git installs only on stable branches (`master`, `main`, `release*`), and lets standalone binaries update from GitHub release assets only when a newer version exists.
- CI: trigger `build-runtime-family-images` on `release_v1` to publish the updated `simple-tcp` runtime image (with `mumble-server` installed) so the Mumble Docker backend integration test can pass.
- Security: suppress CodeQL `py/clear-text-storage-sensitive-data` false-positives in `valve_server.py` where plaintext game-server credentials are written intentionally to Source engine config files read directly by the binary.

2026-04-29
- Security: split schema-backed secret settings (passwords, RCON keys) into a separate `<name>.secrets.json` file written with mode 0o600, keeping plain-text credentials out of the world-readable main data file; the data store transparently merges both files so the rest of the code is unchanged, and `alphagsm data` redacts non-empty secret values rather than printing them.
- Security: add `secret=True` SettingSpec entries for password fields in all remaining legacy game modules (ark, arksurvivalascended, atlasserver, valheim, gravserver, darkandlightserver, projectzomboid, askaserver, bobserver, stationeersserver, arma3headlessserver, terraria vanilla and tshock) so their passwords are routed into the mode-0o600 secrets file instead of the world-readable main data file.
- Packaging: exclude `gamemodules.factorio` from PyInstaller collection so standalone binary builds succeed on all three platforms; the module still raises `NotImplementedError` on import as required by its unit test, but is never needed by the binary since factorio support is unimplemented.

2026-04-28
- Server support: add the TF2 curated mod-support foundation with package-backed module wiring, curated registry metadata for SourceMod and MetaMod, and the first shared desired-state groundwork for server-side mod management.
- Server support: make TF2 curated mod apply persist mod-state errors and expose broken apply state through `status` so failed reconciliations are operationally visible.
- Module loading: treat package-backed canonical modules such as `teamfortress2` as first-class catalog and parity entries so one canonical module id still maps to one import surface.
- Server support: sync readyornotserver queryport and maxplayers into ReadyOrNot/Config/ServerConfig.ini during setup, update, and set.

2026-04-26
- CLI: add a built-in `doctor` command so each server can print its effective runtime decision plus Docker image, container, and local runtime health details.
- Documentation: add a dedicated host-install Docker runtime guide so running AlphaGSM on the host with Docker-backed game servers is documented separately from the manager-container workflow.
- Release prep: stamp standalone binaries with a shared AlphaGSM version, stage release-ready artifacts with SHA256 sidecars, and add a tag-driven GitHub release workflow for Linux, Windows, and macOS builds.
- Docker runtime: make the shared `simple-tcp` family image install `mumble-server` and promote the default Mumble backend lifecycle case to active matrix coverage.

Historical highlights
---------------------

2014
- Initial repository created.

2015
- Built the first AlphaGSM CLI foundation, including command parsing, output multiplexing, and early Minecraft support.
- Renamed the project from sagsm/samanager to AlphaGSM and expanded early documentation.

2016
- Added downloader, settings, backup, activation, and update workflows that shaped the current lifecycle model.
- Expanded Minecraft variants and Steam-oriented support while stabilising the configuration system.

2017
- Continued Steam/public branch integration work and small maintenance merges.

2021
- Refreshed GitHub-facing documentation.

2024
- Resumed active maintenance on the live branch.

2026
- Accelerated platform and runtime work across Windows compatibility, Proton, Docker runtime families, docs/wiki automation, more commands, and broader game-module coverage.
- Hardened CI and release workflows with runtime image publishing, port-collision fixes, and tighter test workflows.
